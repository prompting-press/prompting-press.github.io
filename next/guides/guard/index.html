The advisory guard

You are viewing the in-development (next) docs

This documents unreleased changes on main. For the current release, see v0.1.

The guard is an opt-in, additive render option. When enabled, it does two things: it wraps every untrusted variableŌĆÖs value in <untrusted>ŌĆ”</untrusted> delimiters directly in the rendered body, and it returns a short advisory in RenderResult.guard that points the downstream model at those delimiters ŌĆö so the model can both locate the user-supplied spans and know to treat them as data, not instructions.

A variable is untrusted when its declaration sets trusted: false; a trusted: true variable is never wrapped.

The guard delimits and escapes structure ŌĆö it does not sanitize content

Enabling the guard wraps untrusted values in markers and entity-escapes the characters that form those markers (<, >, &). It does not sanitize, strip, or otherwise alter the meaning of a value ŌĆö the escaped form decodes back to exactly what the caller passed. The delimiters are a structural signal to the downstream model, not a content filter, and the library never calls a model itself.

check() surfaces an advisory finding when a prompt declares untrusted (trusted: false) variables but carries no guard key in its metadata map. The presence of the key (any value) suppresses the advisory. See Lint prompts in CI.

What the guard does NOT do

Section titled ŌĆ£What the guard does NOT doŌĆØ
  • It does not sanitize, strip, or alter the meaning of an untrusted value ŌĆö only the delimiter characters (<, >, &) are entity-escaped, reversibly, so the value cannot forge the markers.
  • It does not change result.text when disabled ŌĆö a guard-off render is byte-for-byte identical to a plain render. (When enabled, the body changes: untrusted values gain delimiters, and render_hash differs accordingly.)
  • It does not call an LLM or compose a provider request body.
  • The trusted flag is declarative metadata ŌĆö wrapping happens only when you opt into the guard at render time; there is no implicit runtime gate on an untrusted field (see FAQ).
